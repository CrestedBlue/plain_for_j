# 요구사항 · 기획

> 상위 인덱스: [plan.md](plan.md) · 관련: [design.md](design.md) · [roadmap.md](roadmap.md)

## 한 줄 정의

여행 계획을 저장·열람·**공유 편집**하는 웹앱. 여러 여행묶음을 저장하고 한 번에 하나를 관리한다.

## 핵심 요구사항

| #   | 동작 | 설명 |
| --- | --- | --- |
| R0  | 여행묶음 관리 | 첫 화면에서 목록을 보고 선택/생성. 한 번에 하나의 묶음 작업 |
| R1  | 여행 일자 선택 | 새 묶음 생성 시 시작일~종료일 선택 → 그 범위의 날짜 나열 |
| R2  | 날짜 → 하루 일정 | 선택한 날짜의 일정 리스트 표시 |
| R3  | 일정 + 지도 | 일정 위치를 지도(현재 SVG 개념지도)에 표시 |
| R4  | 일정 선택 → 상세 + active | 일정 하나 선택 시 상세 메모 + 지도 마커 활성화(양방향 동기화) |
| R5  | 저장·공유 | **서버(MySQL)에 저장 → 누가 접속해도 같은 일정**(v2). (v1은 localStorage였음 → [history.md](history.md)) |

## v2 목표 (풀스택 전환의 동기)

1. **공유 편집** — 사무실에서 같은 일정을 함께 보고 편집. 저장소를 localStorage → 서버 MySQL로 이전.
2. **네이버 장소검색** — 카카오보다 풍부한 국내 장소정보 활용. 서버가 네이버 지역검색을 프록시(secret 서버 보관).
3. **사람 식별** — 누가 만들고 고쳤는지, 사람별 읽음표시가 정확하도록 소셜 로그인 도입.
