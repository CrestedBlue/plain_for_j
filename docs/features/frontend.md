# 프론트 기능

> 상위 인덱스: [../plan.md](../plan.md) · 관련: [backend.md](backend.md) · [../design.md](../design.md)

## 현행 기능 (구현됨)

- 여행묶음 목록·생성(기간 선택), 통합 캘린더 대시보드(일정 없으면 오늘 가운데).
- 일정 CRUD — 조회/수정/추가 3모드 분리(`ScheduleEditor`). 항목 = 장소명·표기명·메모.
- 장소 지정 3경로 (2026-07-11 개편 — 의도별 분리, 파괴적 변경은 항상 명시 버튼으로만):
  - 폼 **장소명 자동완성**(`PlaceAutocomplete`): 입력 시 디바운스 네이버 검색 → 선택하면 이름·위경도를 채움.
  - 지도 하단 **탐색 검색**(`PlaceSearch`): 결과 탭 = 지도에서 위치 확인(폼 불변), `＋` = 새 일정 추가, 편집 중일 때만 "변경"(현재 폼 장소 교체) 노출.
  - **지도 클릭**: 조회 모드 = 탐색(임시 핀 + 근처 후보만, 폼 불변), 추가/수정 모드 = 폼에 좌표 반영.
- Day 탭 고정, 일정 타임라인, 리스트↔지도 active 양방향 동기화.
- 네이버 실지도(`NaverMap`): 번호 마커(단일 색)·경로선(Polyline)·정보창(InfoWindow)·다크 타일 반전, `dayKey` fitBounds·활성 일정 panTo, 탐색/편집 시 점선 임시 핀. (구 SVG 폴백 `VectorMapFallback` 제거됨)
- 모바일 풀스크린(100dvh), 터치 삭제, 320/375 대응.

> **카테고리 표시 제거 (2026-07-11)**: 네이버 지역검색의 자동 카테고리 분류가 부정확(예: 국회의사당→"맛집")해, 검색 태그·조회 카드 뱃지·마커 색·타임라인 점을 모두 제거하고 마커/점은 단일 색으로 통일. 백엔드 `category` 컬럼과 API 필드는 보존(프론트는 저장 시 기본값만 전송, 화면엔 미표시). `categories.ts`의 `CATEGORIES` 메타는 재도입 대비 잔존(현재 미사용).

## 백로그 (미착수)

> 공통 선결: 실제 위경도는 네이버 검색·지도 클릭으로 적재(실지도 핀 위치에 사용), 외부 API는 서버 프록시 경유.

### F1. 일정 추천 (근처 맛집/관광)
- 일정 선택 시 그 위치 근처 추천 → 탭하면 새 일정으로 추가. **선결 충족**(lat/lng + 네이버 검색 프록시 완료) — 남은 건 "선택 일정 근처 자동 추천" 로직뿐. 지도 클릭 탐색은 이미 근처 후보를 띄운다.

### F2. 날씨 (배경 + 애니메이션)
- 예보 가능 기간만, 일자 배경에 비/눈/햇살/구름 CSS 애니메이션(transform/opacity). **데이터: 서버 프록시 권장**, 날짜별 캐싱.

### F4. 공휴일 (Open 캘린더)
- Nager.Date(무키) 또는 공공데이터 특일정보로 공휴일 표시. 추후 국가코드로 해외 확장.

### F5. "스크럼" — 카테고리별 장소 라이브러리
- 여행과 독립된 장소 보관함. 탭 진입 시 지도 메인, 카테고리(식사/카페/관광/숙소)로 분류 저장. 일정 추가 시 라이브러리에서 선택, F1 추천 소스로도 사용. **상위 네비게이션(여행 ↔ 스크럼)** 도입 선행.

### F6. 일정별 의논(댓글) — 프론트 측
- 일정 조회(ViewCard) 하단 댓글 스레드 + 리스트에 `💬`/안 읽음 `!` 뱃지. 평소 화면은 안 복잡, 탭하면 상세에서 의논.
- **선회 사유**: 전역 채팅은 카톡과 겹치고 화면 낭비 → 일정에 묶인 의논으로 전환(맥락에 붙은 결정 기록이 차별점).
- **프로토타입 보류**: 하드코딩 더미 UI = `src/components/dashboard/CommentThread.tsx`(참고 보존), `ScheduleEditor.tsx`에서 주석처리로 비활성. 인증 도입 후 실데이터로 재활성화. 백엔드 측은 [backend.md](backend.md) 참조.
